T003: Switch web storage from localStorage to IndexedDB (unlimited storage)
Blocked By: []
Details:
localStorage is capped at ~5MB by browsers. IndexedDB can hold gigabytes.
Step 1: Create src/lib/indexedDB.ts with a minimal async wrapper: openDB(), idbGet(key), idbSet(key, value), idbGetBytes() (estimates stored size)
Step 2: Update src/lib/storage.ts:
getData() web path: try IndexedDB first; if empty, check localStorage for existing data and auto-migrate it (copy to IndexedDB, then clear localStorage key so stale data doesn't re-appear on next load)
saveData() web path: write to IndexedDB instead of localStorage
getInitialDataSync(): keep as-is (reads localStorage for first-paint on very first load only; cleared after migration so won't show stale data on future loads)
Step 3: Update src/hooks/useLocalStorage.tsx:
Remove the early-return for non-native platforms (lines 113-117) so web ALSO performs the async load from IndexedDB on mount
This ensures IndexedDB data is picked up after migration and on every subsequent load
Step 4: Update src/pages/SettingsPage.tsx:
Make the storage indicator async: use useEffect + idbGetBytes() to compute actual used space
Show "X KB used (no limit)" instead of "X / 5 MB (Y%)" — remove the progress bar since there's no meaningful cap
Files: src/lib/indexedDB.ts (new), src/lib/storage.ts, src/hooks/useLocalStorage.tsx, src/pages/SettingsPage.tsx
Acceptance: Data saves to IndexedDB on web; existing localStorage data migrates automatically on first open; SettingsPage no longer shows "/ 5 MB" cap; app works identically
